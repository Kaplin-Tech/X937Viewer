# X937 Viewer

A self-contained desktop viewer for **X9.37 / Image Cash Letter (ICL)** files —
the Check 21 format banks use to exchange check images. Open a file, browse
every check, see front/back images, inspect all the metadata, and export the
whole file to JSON.

100% local Python. No web server, no network, nothing leaves your machine.

## Features

- **Browse checks and returns** across all cash letters and bundles in a sortable table
- **Search** by amount (`12.34`, `$1,234.56`, or raw cents) or by account number,
  routing number, sequence number, or payee
- **Check images** with front/back toggle, zoom, and fit-to-window (CCITT G4 TIFF decoded via Pillow)
- **Metadata side pane** showing the check detail, addenda A/B/C(/D), image view
  records, and the parent bundle / cash letter / file headers
- **Export to JSON** that is byte-for-byte compatible with
  [moov-io/imagecashletter](https://github.com/moov-io/imagecashletter) output —
  also available from the command line
- **Reads every common on-disk variant**: EBCDIC or ASCII, with or without
  variable-length record prefixes, forward presentment and returns
- **Lenient by design**: malformed files open with warnings instead of crashing

## Install

Requires Python 3.10+.

```
pip install -r requirements.txt
```

(That's just `PySide6` for the GUI and `Pillow` for TIFF decoding.)

## Run

```
python main.py                      # open the GUI
python main.py mycashletter.x937    # open a file directly
```

Or as a module: `python -m x937viewer [file]`.

### Command-line JSON export

```
python -m x937viewer --to-json mycashletter.x937 -o output.json
python -m x937viewer --to-json mycashletter.x937        # to stdout
```

## JSON compatibility

The JSON output mirrors Moov's `imagecashletter` encoding exactly: camelCase
field names, amounts as integer cents, RFC-3339 dates, base64 image data, and
`null` for absent record groups. The test suite parses the sample files in
`Example/` and asserts deep equality — and byte equality — with JSON produced
by Moov's own tooling, so you can mix this viewer into a Moov-based pipeline.

## Tests

```
python -m unittest discover tests -v
```

The acceptance tests compare parser output against reference JSON generated by
Moov's imagecashletter (see `Example/` and `Example/moov/README.md` for
provenance — every reference file is SHA-verified against the upstream repo).

## Project layout

```
x937viewer/
├── records.py      field layouts for every X9.37 record type (data-driven)
├── parser.py       framing/encoding detection, record parsing, hierarchy assembly
├── json_export.py  Moov-compatible JSON serialization
└── gui/            PySide6 desktop UI (table, image viewer, metadata pane)
```
